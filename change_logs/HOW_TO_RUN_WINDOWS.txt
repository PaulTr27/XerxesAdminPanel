============================================================
HOW TO RUN THE XERXES PI ADMIN PANEL — WINDOWS SETUP
============================================================

STEP 1 — Install VirtualBox
-----------------------------
1) Go to: https://www.virtualbox.org/wiki/Downloads
2) Click "Windows hosts" to download the installer
3) Run the installer and click through Next -> Next -> Finish
4) If Windows asks to install network drivers, click Install


STEP 2 — Install Vagrant
--------------------------
1) Go to: https://developer.hashicorp.com/vagrant/downloads
2) Select Windows and download the installer
3) Run the installer and click through Next -> Next -> Finish
4) RESTART YOUR COMPUTER after installation finishes


STEP 3 — Get the Project
--------------------------
Option A — If you have GitHub Desktop:
  1) Open GitHub Desktop
  2) Clone the repository: PaulTr27/XerxesAdminPanel
  3) Note where it saves the folder on your computer

Option B — If you don't have GitHub Desktop:
  1) Go to: https://github.com/PaulTr27/XerxesAdminPanel
  2) Click the green "Code" button
  3) Click "Download ZIP"
  4) Extract the ZIP somewhere easy to find (e.g. Desktop)


STEP 4 — First Time VM Setup
------------------------------
1) Open Command Prompt (search "cmd" in the Start menu)
2) Navigate to the project folder (replace the path with
   wherever you saved it):
     cd "C:\Users\YourName\Desktop\XerxesAdminPanel"
3) Run:
     vagrant up
   This will take several minutes the first time — it
   downloads the Linux environment and installs all tools
   automatically. Just let it run until it finishes.
4) Verify Vagrant is installed correctly by running:
     vagrant --version
   You should see "Vagrant 2.x.x"


STEP 5 — Starting the Website (Every Time)
--------------------------------------------
1) Open Command Prompt

2) Navigate to the project folder:
     cd "C:\Users\YourName\Desktop\XerxesAdminPanel"

3) Start the virtual machine (skip if already running):
     vagrant up

4) Log into the virtual machine:
     vagrant ssh

5) Navigate to the workspace:
     cd /workspace

6) Compile the project:
     make clean
     make

7) Remove any old binary and copy the new one:
     rm /tmp/xerxes_backend
     cp xerxes_backend /tmp/xerxes_backend
     chmod +x /tmp/xerxes_backend

8) Start the server:
     /tmp/xerxes_backend

9) Open your browser (Chrome or Edge) and go to:
     http://localhost:7272

   The admin panel should now be visible and working.


SHUTTING DOWN
--------------
1) In the VM terminal press Ctrl+C to stop the server
2) Type:
     exit
3) Back in your normal Command Prompt run:
     vagrant halt


============================================================
ONE-TIME SERVICE SETUP (do these once after first vagrant up)
============================================================

These steps only need to be done once. The settings survive
vagrant halt / vagrant up. They are lost if you run
vagrant destroy and rebuild the VM from scratch.


SETTING UP SMB FILE SHARING
-----------------------------
Samba is installed automatically by the Vagrantfile, but if
the SMB Start button returns "Unit not found", install it
manually inside the VM:

  1) Log into the VM:
       vagrant ssh
  2) Install Samba:
       sudo apt-get install -y samba
  3) Test it worked — start the server and click SMB Start.
     You should see "smbd started successfully." in the output.


SETTING UP TAILSCALE
---------------------
Tailscale is installed automatically by the Vagrantfile.
Before using Enable/Disable you need to authenticate once:

  1) Log into the VM:
       vagrant ssh
  2) Run:
       sudo tailscale up
  3) Tailscale will print a URL like:
       https://login.tailscale.com/a/XXXXXXXXXX
  4) Open that URL in your browser and log in to your
     Tailscale account (GitHub login works).
  5) Once authenticated the terminal will confirm connection.
     From this point on, the Enable button on the site will
     connect without asking for authentication again.

  NOTE: If the Enable button returns "[no output]" it means
  Tailscale is already connected — that is correct behaviour.
  Use View Status to confirm. You should see a 100.x.x.x IP.


SETTING UP NGROK
-----------------
ngrok is installed automatically but requires a free account
and auth token before it can create tunnels.

  1) Create a free account at: https://ngrok.com
  2) After signing in, go to Your Dashboard -> Your Authtoken
  3) Copy your auth token
  4) Log into the VM:
       vagrant ssh
  5) Run (replace TOKEN with your actual token):
       ngrok config add-authtoken TOKEN
  6) The token is saved inside the VM only — it is not
     stored in the project folder and will never appear
     in the GitHub repository.

  After setup, the Start button will expose port 8080
  (the admin panel backend) to the internet via an ngrok
  tunnel. Use Check Status to confirm it is running.

  NOTE: The ngrok tunnel only runs while the VM is on and
  the process is active. It does not survive a vagrant halt.
  Run Start again each session if you need the tunnel.


============================================================
COMMON ISSUES
============================================================

"No usable default provider could be found"
  -> VirtualBox is not installed. Go back to Step 1.

"vagrant: command not found"
  -> Vagrant is not installed or you haven't restarted
     your computer. Go back to Step 2.

"Permission denied" when running the server
  -> Make sure you are running from /tmp/ not /workspace/
     Follow Step 7 exactly — the rm, cp, and chmod
     commands must all be run before starting the server.

"Vagrant up takes forever / gets stuck"
  -> This is normal the very first time. It is downloading
     a full Linux system. Leave it running and wait.

"Unit smbd.service not found" when clicking SMB Start
  -> Samba was not installed during provisioning.
     See SETTING UP SMB FILE SHARING above.

"Tailscale Enable returns [no output]"
  -> Tailscale is already connected. This is correct.
     Click View Status to confirm — you should see a
     100.x.x.x IP address in the output.

"Tailscale Enable keeps asking to authenticate"
  -> The timeout killed the process before auth completed.
     Do the one-time setup via SSH terminal instead.
     See SETTING UP TAILSCALE above.

"ngrok Start button returns an error"
  -> Your auth token has not been configured in the VM yet.
     See SETTING UP NGROK above.
