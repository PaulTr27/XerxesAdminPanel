SUMMARY
-------
Implemented the Network and Tunnels sections fully.
Added hostname rename, SMB file sharing controls, Tailscale
tunnel management, and ngrok tunnel controls. Backend extended
with new API endpoints, service status badges, and a hardened
ALLOWLIST covering all new commands. Fixed a compiler warning

FILES MODIFIED

src/system_ops.cpp
  - Fixed _GNU_SOURCE redefinition compiler warning by wrapping
    the define in an #ifndef guard and moving it before all
    includes so it takes effect before system headers are read
  - Extended ALLOWLIST with all new service and tunnel commands:
      samba_start     -> sudo systemctl start smbd 2>&1
                         (appends success/fail message since
                          systemctl is silent on success)
      samba_stop      -> sudo systemctl stop smbd 2>&1
                         (appends success/fail message)
      samba_status    -> SYSTEMD_COLORS=0 systemctl status smbd
                         --no-pager --full 2>&1
                         (SYSTEMD_COLORS=0 forces plain text
                          output through non-TTY pipes)
      tailscale_up    -> timeout 10 sudo tailscale up 2>&1
                         (capped at 10 s; outputs auth URL if
                          authentication is needed, otherwise
                          confirms up)
      tailscale_down  -> sudo tailscale down 2>&1
      tailscale_status-> tailscale status 2>&1
      ngrok_start     -> ngrok http 8080 run in background,
                         logs to /tmp/ngrok.log
      ngrok_stop      -> pkill ngrok
      ngrok_status    -> pgrep -x ngrok check
      reboot          -> sudo systemctl reboot 2>&1
      shutdown        -> sudo systemctl poweroff 2>&1
  - Added set_hostname() — validates proposed name strictly
    against RFC 1123 rules (alphanumeric + hyphens only,
    1-63 chars, no leading/trailing hyphens) before building
    the shell command. Also updates the 127.0.1.1 entry in
    /etc/hosts via sed to prevent sudo hostname warnings
  - Added get_service_status() — uses a separate internal
    hardcoded map to check service states safely:
      smbd      -> systemctl is-active smbd
      tailscaled-> tailscale status, greps for IP line to
                   reflect connection state rather than daemon
                   state (daemon stays active after tailscale
                   down, connection state does not)
      ngrok     -> pgrep -x ngrok check

BUGS FIXED

- _GNU_SOURCE redefined compiler warning on every make
  (GCC on Debian 13 defines it implicitly at command-line
  level; source-level define was always a redefinition)

- SMB badge showing "inactive inactive" instead of "inactive"
  systemctl is-active already prints "inactive" when a unit
  is not found and exits non-zero, so the || echo inactive
  fallback was appending a second "inactive" line. Removed
  the fallback for smbd and tailscaled status checks.

- Tailscale badge staying "active" after pressing Disable
  systemctl is-active tailscaled reflects daemon state, not
  connection state. The daemon stays running after tailscale
  down. Fixed by switching to a tailscale status grep that
  checks for a connected IP address line instead.

- Start/Stop commands showing [no output]
  systemctl start and stop are intentionally silent on
  success. Fixed by appending explicit confirmation strings
  to samba_start, samba_stop, and tailscale_up so the user
  always receives feedback in the output panel.
